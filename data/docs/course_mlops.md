# MLOps 엔지니어 과정 이해 자료

## 왜 지금 MLOps 엔지니어인가?
- 모델을 만드는 것은 쉬워도 운영이 어렵다.
- 국내 대기업들이 겪는 공통된 공백이 존재.
- 데이터 파이프라인 및 모델 운영 자동화를 통합한 역량 수요 증가 중.

## 학습 흐름 타임라인
| **STEP**   | **시간**  | **내용**                   |
|-----------|----------|---------------------------|
| STEP 1    | 184h     | 데이터 엔지니어링 기본기         |
| STEP 2    | 80h      | AI Ready Data 구축           |
| STEP 3    | 248h     | 분산 처리 실시간 시스템 설계      |
| STEP 4    | 144h     | MLOps 시스템 자동화           |
| STEP 5    | 304h     | 실무 프로젝트 CI/CD/CT 통합   |

---

## 수료 시 확보하는 결과물 카드
- **Lv.1**: 데이터 설계 – SQL·NoSQL 기반 데이터 파이프라인 구축 프로젝트.
- **Lv.2**: 대용량 데이터 – Spark·Kafka 기반의 실시간 로그 통합 파이프라인.
- **Lv.3**: 모델 운영 시스템 – MLflow·Airflow·Docker·K8s 기반의 모델 생애 주기 자동화.

---

## 시장 근거
- **2025년**: 행정안전부가 공공 데이터의 데이터 포맷과 품질 기준을 마련.
- **대기업 투자**: 현대자동차, 삼성 등의 데이터 인프라에 대한 수백억 원 투자.

## 차별점
- MLOps 과정은 데이터 엔지니어링부터 자동화까지 통합된 커리큘럼을 제공하여 다양한 실무 프로젝트 경험을 쌓을 수 있음.

## 핵심 기술 스택
- **프로그래밍**: Python, pandas, Git/GitHub
- **데이터베이스**: MySQL, MongoDB, Redis
- **분산 처리**: Apache Spark, Apache Kafka
- **클라우드 인프라**: AWS S3, EKS
- **운영**: scikit-learn, MLflow, FastAPI
- **워크플로우 자동화**: Airflow, Docker, Kubernetes
