# AI 오케스트레이션 과정 브리핑 카드

| 항목          | 내용                                                                      |
|---------------|---------------------------------------------------------------------------|
| 공식과정명    | Auto-Healing AI 멀티에이전트워크플로우기반 오케스트레이션 애플리케이션 개발자 양성과정 |
| 상담용축약명  | AI 오케스트레이션과정                                                   |

| 한줄정의      | 내 상황에 맞는 에이전트를 설계하는 과정                                 |
| 핵심질문      | 여러 에이전트를 어떻게 협업시켜 일을 수행하게 만들 것인가                |

| 공식직무      | AI, API, 애플리케이션 개발자 모델을 활용한 웹 앱 서비스 구현             |

## 핵심 가치 축

1. **멀티에이전트오케스트레이션**
   - LangGraph, Function Calling AI, LLM 기반 여러 에이전트 협업 구조 설계
   - 단일 에이전트를 넘어 에이전트 팀 구성

2. **자가복구에이전트**
   - 에이전트가 에러를 감지하고 스스로 복구
   - 실제 서비스 환경 필수 역량의 중점 교과

## 추천 대상

- 서비스 기획 운영 에이전트 설계에 관심 있는 분
- 비전공자 직무 전환형 지원자 친화적
- 코딩 잘하는 사람이 아닌 도구로 쓰는 사람이 되고 싶은 분

## 과정 이해자료

### 왜 지금 오케스트레이션 개발자인가

- 2026년부터 에이전트 도입 본격화
- 대기업 업무 자동화에 에이전트 투입 확대
- 시장에서 찾는 사람: 내 업무 산업에 에이전트를 적용할 수 있는 사람

## 차별점

| 영역 | 이 과정의 특징                                        |
|------|-----------------------------------------------------|
| 진입방식 | Vibe Coding, 노코드 비전공자 친화적               |
| 핵심포커스  | 멀티에이전트오케스트레이션, 단일 활용 아님              |
| 특별화         | 자가복구 에이전트, 실무 필수역량의 중점 교과 |

### 실무 적용 시나리오

1. **리서치 분석 보고서 작성 에이전트 협업 시스템**
   - 여러 에이전트가 역할을 나누어 리서치부터 보고서 작성까지 자동화
  
2. **고객 응대 연동 멀티에이전트**
   - 고객 문의 분류 → 응대 → CRM 기록까지 에이전트 협업
  
3. **내부 문서 기반 업무 자동화 에이전트**
   - 기업 내부 문서를 활용한 업무 처리 자동화 시스템

## 마무리

이 과정은 단순한 코딩 부트캠프가 아닙니다. 내 업무에 맞는 에이전트를 설계할 수 있는 사람을 양성하는 과정이에요. 비전공자 직무 전환자에게 잘 맞는 과정입니다. 

960 시간 뒤, 지원자는 코딩을 잘하는 사람이 아니라 내 상황에 맞는 에이전트를 설계할 수 있는 사람이 될 것입니다.
