# 상담 챗봇 프로젝트 기획 정리.

# 1. 프로젝트 개요

## 프로젝트 목적

신규 교육 프로그램(BEYOND SW 캠프) 관련 정보를 학생들에게 안내하는 문서 기반 상담 챗봇 구축

학생들이 질문하면:

- 프로그램 안내
- 지원 대상
- 커리큘럼 설명
- 운영 정책
- 환불 규정
- 신청 방법
  등을 자연스럽게 답변하는 형태

---

# 2. 프로젝트 방향

현재 프로젝트는 단순 FAQ 챗봇이 아니라:

> 문서 기반 안내 챗봇

구조에 가까움

즉:

- 정적인 문서 데이터를 기반으로
- GPT가 문맥을 이해하여
- 자연스럽게 답변하는 구조

실시간 외부 데이터 연동은 하지 않음

---

# 3. 서비스 구조

## 서비스 흐름

기존 홈페이지에 챗봇 진입 버튼 또는 아이콘 배치

사용자가 버튼을 클릭하면 챗봇 전용 페이지로 이동하여 상담 진행

---

## 전체 구조

기존 회사 홈페이지
↓
챗봇 진입 버튼 클릭
↓
챗봇 전용 페이지 이동
↓
사용자 질문 입력
↓
FAQ(JSON) 우선 검색
↓
FAQ에 없으면 Markdown 문서 검색
↓
OpenAI API 호출
↓
답변 생성 및 출력

---

# 4. 프로젝트 특징

- 문서 기반 안내 챗봇
- FAQ + 문서 검색 혼합 구조
- GPT 기반 자연어 응답
- 관리자 열람 가능
- 상담 기록 저장
- 개인정보 암호화 저장
- 기존 홈페이지와 연동

---

# 5. 데이터 구조 방향

## 최종 구조

Markdown(MD) + FAQ(JSON) 혼합 구조

---

## Markdown(MD)

설명형 문서를 저장

### 관리 대상 예시

- 과정 소개
- 지원 대상
- 커리큘럼
- 운영 정책
- 프로젝트 안내
- 환불 규정

### Markdown 예시

# 과정 소개

## 지원 대상

- 대학생
- 취업 준비생

## 수강 기간

8주 과정

## 교육 방식

온라인/오프라인 수업 및 프로젝트 진행

---

## FAQ(JSON)

반복적으로 들어오는 질문을 저장

### 관리 대상 예시

- 수강료
- 개강일
- 신청 링크
- 준비물
- 문의 방법

### FAQ(JSON) 예시

[
{
"category": "수강료",
"questions": [
"수강료는 얼마인가요?",
"무료인가요?"
],
"answer": "수강료는 무료입니다."
}
]

---

# 6. 응답 처리 흐름

사용자 질문
↓
FAQ(JSON) 우선 검색
↓
FAQ 존재 시 즉시 응답
↓
FAQ 없을 경우 Markdown 문서 검색
↓
관련 문서 내용을 GPT에게 전달
↓
GPT가 자연스럽게 답변 생성

---

# 7. 답변 범위

챗봇은 제공된 문서에 포함된 내용만 답변한다.

문서에 없는 내용이나 확인되지 않은 정보는 임의로 답변하지 않고 담당자 문의를 안내한다.

---

## 답변 가능 범위

- 교육 과정 안내
- 지원 대상
- 커리큘럼
- 운영 정책
- 신청 방법
- 수강 관련 안내
- 환불 및 문의 안내

---

## 답변 제한 범위

- 문서에 없는 정보
- 교육 과정과 무관한 질문
- 개인정보를 과도하게 요구하는 질문
- 욕설, 장난성 질문, 부적절한 질문

---

# 8. 모르는 질문 처리 방식

문서에서 확인되지 않는 질문은 답변을 생성하지 않고 담당자 문의를 안내한다.

### 예시 응답

“해당 내용은 현재 제공된 안내 문서에서 확인되지 않습니다. 정확한 안내를 위해 담당자에게 문의해 주세요.”

교육 과정과 관련 없는 질문이나 이상한 질문은 처음부터 관련 질문만 하도록 안내한다.

### 예시 응답

“저는 교육 과정 안내를 위한 상담 챗봇입니다. 과정 신청, 커리큘럼, 운영 방식, 문의 방법 등과 관련된 질문을 입력해 주세요.”

---

# 9. 상담 연결 방식

챗봇이 답변하지 못하는 경우 사용자가 담당자에게 직접 문의할 수 있도록 연결 수단 제공

## 상담 연결 수단

- 이메일
- 전화번호
- 담당자 문의 링크

---

# 10. 개인정보 처리 기준

학원/교육 과정 특성상 상담 기록 저장 필요

## 기본 방향

- 상담 기록 저장
- 개인정보 암호화 저장
- 관리자 페이지에서 열람 가능
- 개인정보 수집 및 이용 목적 안내
- 보관 기간 경과 시 파기

---

## 수집 가능 정보 예시

- 이름
- 연락처
- 이메일
- 질문 내용
- 답변 내용
- 상담 일시

---

## 개인정보 보호 기준

- 개인정보 암호화 저장
- 관리자 권한 기반 열람
- 불필요한 개인정보 수집 제한
- 보관 기간 경과 시 파기

---

# 11. 상담 기록 저장 여부

상담 기록은 저장한다.

익명 상담은 현실적으로 어려우므로 전체 상담 내용을 저장하되 개인정보 데이터는 암호화하여 관리한다.

---

## 저장 항목 예시

- 사용자 식별 정보
- 질문 내용
- 챗봇 답변
- 상담 일시
- 문의 유형
- 담당자 연결 여부

---

## 관리자 페이지 기능

- 상담 기록 조회
- 사용자별 상담 내역 확인
- 질문 유형 분석
- 답변 실패 사례 확인
- 담당자 문의 필요 건 확인

---

# 12. 개인정보 보관 기간

개인정보는 원칙적으로 수집 및 이용 목적이 달성되면 파기한다.

초기 MVP에서는 회사 개인정보 처리방침 및 내부 기준에 맞춰 보관 기간을 결정한다.

## 임시 제안

- 상담 기록 보관 기간: 1년 검토
- 보관 기간 경과 후 자동 또는 수동 파기

# 13. 추천 질문 버튼

초기 화면에 추천 질문 버튼 제공

추천 질문 데이터는 FAQ JSON 기반으로 관리

---

## 목적

---

# 14. 문서 업데이트 방식

초기 MVP에서는 개발자가 Markdown 문서와 FAQ JSON 파일을 직접 수정 후 배포하는 방식으로 운영

---

## 관리 대상

- Markdown 문서
- FAQ JSON
- 추천 질문 데이터

---

## 중요 조건

챗봇은 배포 중에도 중단되면 안 된다.

따라서 무중단 배포 또는 최소한의 서비스 중단 방식 검토 필요

---

# 15. 응답 스타일

챗봇은 친절한 상담원 톤의 존댓말 사용

문서 기반 내용을 요약하여 답변

---

## 응답 기준

- 존댓말 사용
- 친절한 상담원 톤
- 문서 기반 답변만 제공
- 근거 없는 내용 금지
- 답변은 5문장 이내
- 한 번의 답변 안에 필요한 내용을 모두 포함
- 여러 번에 나누어 출력하지 않음

---

# 16. 비용 관리 및 모델 선정

챗봇은 OpenAI API 기반으로 운영

상담 챗봇 특성상 토큰 제한은 현실적으로 어려움

따라서 FAQ 우선 응답 및 문서 검색 최적화를 통해 비용 관리 진행

---

## 비용 관리 방향

- FAQ JSON 우선 응답
- FAQ 처리 가능 질문은 GPT API 호출 최소화
- Markdown 문서는 관련 부분만 검색하여 GPT 전달
- 답변 길이는 5문장 이내 제한
- 가성비 좋은 모델 우선 선정

---

## 모델 선정 기준

- 문서 기반 상담에 충분한 성능
- 빠른 응답 속도
- 낮은 API 비용
- 안정적인 한국어 응답
- 운영 비용 예측 가능성

---

## 모델 후보

초기 MVP 기준:

- GPT-5.4 mini 우선 검토

선정 이유:

- 비용 효율
- 빠른 응답 속도
- 상담 챗봇 수준에 적합한 성능
